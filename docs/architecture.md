# UX Notes

## Page Model

The prototype direction is built around a cafe visitor path:

1. Feel the atmosphere.
2. See the signature products.
3. Understand menu categories.
4. Trust the location and visit details.
5. Move toward enquiry or visit intent.

## Interface Priorities

| Priority | Reason |
|---|---|
| Scroll-led reveal | Supports visual storytelling without overwhelming first load |
| Product focus | Drinks and desserts sell the experience |
| Media optimization | Visual-heavy sites need disciplined assets |
| Mobile clarity | Cafe discovery often happens on mobile |
| Public-safe repo | Browser profiles and private captures must stay out |

## Execution Signal

The work shows the ability to plan visual UX, prepare media-heavy prototypes, and publish only the recruiter-safe summary.
